# Валидация naming-кандидатов

Этот документ превращает naming checklist из identity package в операционный фильтр для будущих naming-сессий.

## Задача документа
Любой новый naming candidate должен проходить через один и тот же набор критериев, чтобы бренд не смещался в сторону generic habit tracker, wellness app или соцсети достижений.

## Рекомендуемый scoring rubric
Оцени каждый критерий по шкале `0-5`. Максимум: `25`.

| Критерий | Что проверяем | Вес |
| --- | --- | --- |
| Specificity | Имя конкретное, а не расплывчатое | 5 |
| System fit | Имя поддерживает систему proof / review / progress | 5 |
| Pronunciation | Легко произносится по-английски | 5 |
| Russian fit | Естественно звучит для русскоязычного пользователя | 5 |
| Expansion room | Не запирает продукт в side projects или narrow use case | 5 |

### Порог принятия
- `21-25` — сильный кандидат, можно двигаться дальше
- `16-20` — возможно, но нужен дополнительный разбор
- `0-15` — отклонять

## Жёсткие критерии отклонения
Имя нужно отклонить сразу, если оно:
- звучит как generic habit tracker
- звучит как wellness или self-help product
- звучит как social network for achievements
- слишком абстрактное или метафорическое без системы
- не работает в коротком UI-копирайте
- плохо произносится на английском
- ломает system-oriented visual identity
- привязано только к узкому future scenario

## Практическая проверка
Перед финальным решением проговори имя в пяти контекстах:
1. Название продукта в header.
2. Заголовок карточки.
3. Кнопка или CTA.
4. Пустое состояние.
5. Уведомление.

Если имя выглядит чужеродно хотя бы в двух из пяти контекстов, это плохой знак.

## Примеры приемлемых имён
Эти примеры соответствуют текущему identity package и его логике.

| Имя | Почему подходит |
| --- | --- |
| `Checkpoint` | Конкретное, системное, хорошо ложится на proof/review/progress |
| `Proofline` | Сохраняет связь с proof и ощущение структурной линии |
| `Commit Signal` | Передаёт действие и подтверждение, звучит операционно |

## Примеры неприемлемых имён
Эти примеры показывают, что именно надо отсекать.

| Имя | Почему отклонить |
| --- | --- |
| `HabitHive` | Слишком похожо на habit tracker и soft productivity app |
| `GlowUp` | Wellness / lifestyle ассоциация вместо системы ответственности |
| `WinStreak` | Слишком gamified и achievement-feed по ощущению |
| `BuddyBoost` | Слишком лёгкое, разговорное и не системное |
| `Progressly` | Generic SaaS звучание без брендовой плотности |

## Итоговая рекомендация
Если кандидат не проходит по specificity, system fit и anti-generic checks одновременно, его не следует продвигать дальше даже при хорошей звучности.
