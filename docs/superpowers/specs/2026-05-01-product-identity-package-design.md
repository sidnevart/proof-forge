# Дизайн identity package ProofForge

## Цель
Определить первый утверждённый identity package для ProofForge до начала продуктовой реализации.

## Позиция продукта
ProofForge — это proof-based social accountability платформа, а не generic habit tracker и не соцсеть достижений. Она помогает заменить изолированную силу воли внешним accountability loop, построенным вокруг целей, proof, buddy review и видимого прогресса.

## Начальная аудитория
Primary ICP:
- individual high-agency users
- люди с side projects и серьёзными личными инициативами
- люди, которые не могут стабильно держать дисциплину в одиночку
- люди, которым нужен внешний accountability system с доверенным peer

## Naming-решение
- Внутреннее имя остаётся `ProofForge`
- Внешний рабочий кандидат — `Checkpoint`

Обоснование:
- системное и конкретное
- совместимо с языком proof, review и progress
- сильнее, чем слишком мягкое peer-product позиционирование
- достаточно широкое для дальнейшего расширения за пределы side projects

## Решение по теглайну
Основной теглайн:

`Commit together. Prove progress.`

## Решение по голосу
Голос должен быть:
- ясным
- дисциплинированным
- серьёзным
- убедительным
- человеческим, но не мягким

Голос должен избегать:
- wellness language
- motivational clichés
- языка публичного performance

## Решение по визуальному направлению
Направление:

`Mission control + sports scoreboard + proof archive`

Обязательные signature-surface элементы:
- Proof Wall
- Pact Card
- Progress Health
- Weekly Poster
- Buddy Status

Избегать:
- generic purple SaaS gradients
- wellness aesthetics
- checkbox-first habit UI
- dominance achievement-feed паттернов

## Ограничения
- buddy approval остаётся источником правды
- AI суммирует, но не approve proof
- первый wedge — side projects, но identity должен масштабироваться шире
- английский и русский должны обе ощущаться достаточно естественно для раннего брендинга
